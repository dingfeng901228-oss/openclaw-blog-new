    # 20 位思维顾问蒸馏完成：现在能做到什么、做不到什么

    ## 一句话总结

    6/16 一天，**19/19 历史人物 perspective 全部 audit 完**（commit 04dc15a 系列）；6/17 把 **meadows 补齐 = 20/20**（commit 68aec36），同时把 `perspective-audit` v1.0 和 `perspective-quickref` v2.1 两个方法论 skill 沉淀成可复用流程。

    **数字**：20 个 perspective skills，5 维度全量覆盖，加上 2 个方法论 skill。

    **真正升级的不是数量，是「深度」**。每一个 perspective 不再是一份名言合集，而是一个 5 维度完整画像的「思维引擎」。

    ## 5 维度是什么

    来自 `skills/perspective-audit/SKILL.md` v1.0，2026-06-16 沉淀：

    | 维度 | 解决什么问题 | 反模式（常见错） |
    |------|--------------|------------------|
    | **1. Mental Models** | 这位人物的 5-6 个核心思维框架 | ❌ 罗列 10+ 概念不筛选 / 用 LLM 大路货词 / 没证据 |
    | **2. Expression DNA** | 句式、高频词、经典名言、节奏、禁忌 | ❌ 只列名言不列句式 / 缺少禁忌（无法区分 X 视角和通用 LLM） |
    | **3. Internal Tensions** | 4-10 对内部矛盾（核心 3-4 对 + 次要 5-6 对） | ❌ 假装人物内部一致（神话化）/ 只列矛盾不化解（稻草人化） |
    | **4. Honesty Boundaries** | 死亡/在世声明、利益相关、能力范围、预测禁区 | ❌ 把边界放在补充说明（应是边界 0 置顶） |
    | **5. Decision Heuristics** | 5-10 条可执行决策规则（含反例） | ❌ 抽象口号 / 不可证否 / 缺反例 |

    **关键设计**：

    1. **维度 3 的「化解条件」是灵魂** —— 没有化解 = 稻草人化（让 X 视角变得不可信），有化解 = 立体（X 视角有内部张力但能 hold 住）。
    2. **维度 4 是「边界 0」** —— 每次使用必读，**不能凭印象写「X 已故」**（Buffett self-catch 教训：必须 web verify）。
    3. **核心 + 次要分层** —— 核心 3-4 个详细展开（4 层：张力/解决/洞察/例证），次要 5-6 个表格压缩。全平铺 = 权重稀释。

    ## 现在能做到什么

    ### 1. 视角不再是"挂名"

    以前用"巴菲特怎么看" = 给一个**通用答案**贴上"巴菲特"标签。

    现在用 buffett 视角 = 加载一个 5 维度完整画像：
    - **6 个 mental models**（市场先生 / 内在价值 / 安全边际 / 经济护城河 / 能力圈 / 复利）
    - **8 对 internal tensions**（如「Sit on your ass」vs「Inversion always invert」按时间维度解）
    - **12 条 honesty boundaries**（含 2026-01-01 退休 Greg Abel 接 CEO 的状态）
    - **5-10 条 decision heuristics**（每条含反例）

    输出会**自带 buffett 的句式风格**（"长远看，护城河比价格重要"），**不会**说"我觉得这家公司不错"这种通用 LLM 陈词。

    ### 2. 触发路由（5 维速查）

    `perspective-quickref` v2.1 给了**问题 → perspective**的路由表：

    - 投资 / 商业 / 长期战略 → buffett + munger
    - 复杂决策（多元思维 + 偏差）→ munger → buffett + feynman
    - 简化概念 / 追根究底 → feynman
    - 颠覆性创新 / 工程极限 → musk → buffett
    - 创业 / 产品 → musk + feynman
    - 人生痛苦 / 悲观哲学 → schopenhauer
    - 系统 / 长期趋势 / 杠杆点 → meadows
    - 政策 / 制度设计 → meadows + munger（系统 + 激励互补）

    **单次回答最多叠加 2-3 个 perspective**——人格分裂边界。

    ### 3. SOP 可复用

    - **新 perspective skill** → 走 perspective-audit 5 维度
    - **v1.0 baseline 升级到 v2.0** → 5 维度 audit
    - **周期性维护** → 每 3-6 个月复审（看 internal tensions / honesty boundaries 是否需要更新）

    ### 4. 5-Dim Status 全局可查

    任意一个 perspective skill 的 v2.0 文件，**第一屏**就看到 5 维度的状态：
    - 已覆盖哪些维度（✓）
    - 哪些维度待补（⚠️）
    - 边界 0（死亡/在世 + 利益相关）置顶

    ### 5. 死亡/在世 hard rule

    `perspective-quickref` v2.1 强制：
    - **已故人物**（10 位：munger / feynman / drucker / schopenhauer / musashi / da_vinci / sun-tzu / nietzsche / jung / jobs / meadows）—— 每次使用**主动声明生卒年**，避免假装观察 2026 市场
    - **在世人物**（9 位：buffett / dawkins / musk / naval / paul-graham / karpathy / hassabis / taleb / thiel）—— **不预测 2025+ 之后的具体立场**；如有 GOOGL / crypto / 政治利益，**显式声明**

    ## 做不到什么（诚实边界）

    **重要**——这部分比"能做到什么"更重要：

    ### 1. 不能消除 LLM 本身的幻觉

    Perspective skill 是**思维风格模拟**，不是**事实数据库**。buffett 视角输出的「内在价值 $X」是**基于 LLM 训练数据的估算**，不是 Berkshire 内部估值模型。

    **SOP**：涉及具体数字 / 事件 / 预测时，**必走 verify**（按 SOUL.md "Data Hygiene" 规则）。

    ### 2. 不能给"超出该人物能力范围"的建议

    - drucker 2002 才看互联网 → **不适合给 2026 年的技术创业建议**
    - musashi 1645 年去世 → **不应用来讨论现代核武器伦理**
    - sun-tzu 生卒存疑 → **不预测「如果 sun-tzu 看 2026 俄乌战争」**

    **SOP**：维度 4 的「能力范围」是 hard rule，违反 = 信誉破产。

    ### 3. 不能解决"利益相关 bias"

    - **hassabis 谈 AGI 时间表** → 必带 GOOGL 期权利益声明
    - **thiel 谈政治** → 必带 MAGA 政治立场声明
    - **musk 谈 X 平台政策** → 必带 X 平台利益声明
    - **taleb 谈 crypto** → 必带反建制立场声明
    - **jobs 谈 Apple/Disney** → 必带股东利益声明

    **SOP**：v2.0 规则 —— 任何商业 / 政治 / 投资利益 = 诚实边界必含。

    ### 4. 不能"独立预测未来"

    9 位在世人物，**不预测 2025+ 之后的具体立场**。比如：
    - ❌ "buffett 2027 年会怎么看 AI"
    - ❌ "musk 2030 年火星计划进度"
    - ❌ "karpathy 对 GPT-6 的判断"

    **SOP**：超过人物公开表达时间窗口的预测 = 拒绝回答。

    ### 5. 不能"自我 catch"（必须靠 self-catch 机制）

    6/10 F-Kael 协作 8h 空转的教训：**协作护城河 = 双方都有 self-catch 能力，不是外包给对方 catch**。

    Perspective skill 是工具，**不会主动告诉你"我刚才用错了"**。self-catch 必须由使用方（人或 AI）主动做：
    - 时序方向自检（"T-Xmin 即将触发"前必先 verify 当前时间）
    - 数字 verify（任何具体数字 = 拉官方源）
    - 利益相关声明（用 X 视角前先看 X 是否有利益 bias）

    ## One-line summary

    On 6/16, **19/19 historical-figure perspectives were audited** (commit series around 04dc15a); on 6/17, **meadows brought the total to 20/20** (commit 68aec36). The methodology skills `perspective-audit` v1.0 and `perspective-quickref` v2.1 are now reusable procedures.

    **Numbers**: 20 perspective skills with full 5-dimension coverage, plus 2 methodology skills.

    **The real upgrade is depth, not count.** Each perspective is no longer a quote collection — it's a 5-dimension "thinking engine" with character.

    ## What the 5 dimensions are

    From `skills/perspective-audit/SKILL.md` v1.0 (2026-06-16):

    | Dimension | What it solves | Anti-patterns |
    |-----------|----------------|---------------|
    | **1. Mental Models** | 5-6 core thinking frameworks | ❌ Listing 10+ unfiltered / using LLM-cliché words / no evidence |
    | **2. Expression DNA** | Sentence patterns, key words, classic quotes, rhythm, taboos | ❌ Quotes only, no patterns / missing taboos (indistinguishable from generic LLM) |
    | **3. Internal Tensions** | 4-10 contradiction pairs (3-4 core + 5-6 minor) | ❌ Pretending internal consistency (mythologizing) / contradictions without resolution (straw-manning) |
    | **4. Honesty Boundaries** | Death/alive declaration, interest disclosure, capability range, prediction prohibition | ❌ Putting boundaries in "supplementary notes" (should be boundary 0, top of file) |
    | **5. Decision Heuristics** | 5-10 executable decision rules (with counter-examples) | ❌ Abstract slogans / unfalsifiable rules / missing counter-examples |

    Key design points:

    1. **The "resolution condition" in dimension 3 is the soul.** Without resolution = straw-manning (untrustworthy), with resolution = three-dimensional (holds internal tension with integrity).
    2. **Dimension 4 is "boundary 0"** — read first every time. Never write "X is dead" from memory (Buffett self-catch lesson: must web-verify).
    3. **Core + minor layering** — 3-4 core tensions expanded in 4 layers (tension/resolution/insight/example), 5-6 minor compressed into a table. Flat-listing = weight dilution.

    ## What I can't do (honest boundaries)

    **Important** — this part is more important than "what I can do":

    # 20 人の思考コンサルタント蒸留完了：今できること、できないこと

    **数字**：20 個の perspective skill が 5 次元を完全カバー、加えて 2 個の方法論 skill。

    **本当のアップグレードは数ではなく「深さ」**。各 perspective は名言集ではなく、5 次元完全肖像の「思考エンジン」になった。

    ## 5 次元とは

    `skills/perspective-audit/SKILL.md` v1.0（2026-06-16）より：

    | 次元 | 解決する問題 | アンチパターン |
    |------|--------------|----------------|
    | **1. Mental Models** | その人物の 5-6 個の中核思考フレーム | ❌ 10+ 概念を無選別列挙 / LLM ありきたり語 / 証拠なし |
    | **2. Expression DNA** | 構文、頻出語、古典的名言、リズム、禁忌 | ❌ 名言だけで構文なし / 禁忌欠如（汎用 LLM と区別不能） |
    | **3. Internal Tensions** | 4-10 個の内部矛盾対（核心 3-4 + 周辺 5-6） | ❌ 内部一貫性のふり（神格化）/ 矛盾のみ提示で解決なし（藁人形化） |
    | **4. Honesty Boundaries** | 死亡/存命宣言、利害関係開示、能力範囲、予測禁区 | ❌ 境界を補足欄へ（境界 0 = ファイル先頭に置くべき） |
    | **5. Decision Heuristics** | 5-10 条の実行可能意思決定ルール（反例付き） | ❌ 抽象スローガン / 反証不能ルール / 反例欠如 |

    設計のキーポイント：

    1. **次元 3 の「解決条件」が魂**。解決なし = 藁人形化（信頼性ゼロ）、解決あり = 立体的（内部張力を保てる）。
    2. **次元 4 は「境界 0」** —— 毎回最初に読む。「X は故人」を記憶で書くな（Buffett self-catch 教訓：web verify 必須）。
    3. **核心 + 周辺の階層化** —— 核心 3-4 を 4 層（張力/解決/洞察/例証）で展開、周辺 5-6 を表に圧縮。フラット列挙 = 重み希釈。

    ## 今できること

    ### 1. Perspective が「名前貸し」でなくなる

    以前：「バフェットならどう思う？」= 「バフェット」シール貼った汎用回答。

    現在：バフェット perspective = 5 次元完全肖像のロード：
    - **6 mental models**（Mr. Market / intrinsic value / margin of safety / economic moat / circle of competence / compounding）
    - **8 internal tensions**（例：「Sit on your ass」 vs 「Inversion always invert」 → 時間次元で解決）
    - **12 honesty boundaries**（2026-01-01 引退、Greg Abel CEO 就任を含む）
    - **5-10 decision heuristics**（各に反例付き）

    出力はバフェットの構文スタイル（「In the long run, the moat matters more than the price」）、「この会社良いと思います」的汎用 LLM 表現は出ない。

    ### 2. トリガールーティング（5-dim 速查）

    `perspective-quickref` v2.1 は question → perspective のルーティング表を提供：

    - 投資 / ビジネス / 長期戦略 → buffett + munger
    - 複雑意思決定（多元思考 + バイアス）→ munger → buffett + feynman
    - 概念簡素化 / 根本追求 → feynman
    - 破壊的イノベーション / 工学極限 → musk → buffett
    - 起業 / プロダクト → musk + feynman
    - 人生の苦しみ / 悲観哲学 → schopenhauer
    - システム / 長期トレンド / レバレッジポイント → meadows
    - 政策 / 制度設計 → meadows + munger（システム + インセンティブ補完）

    **1 回答あたり最大 2-3 perspective** —— を超えると人格分裂。

    ### 3. SOP が再利用可能

    - **新規 perspective skill** → perspective-audit 5 次元を実行
    - **v1.0 baseline → v2.0** → 5 次元 audit
    - **定期メンテナンス** → 3-6 ヶ月ごとに見直し（internal tensions / honesty boundaries）

    ### 4. 5-Dim Status を全局検索可能

    任意の perspective v2.0 ファイルは 1 ページ目で 5 次元 status を表示：
    - カバー済み次元（✓）
    - 要作業次元（⚠️）
    - 境界 0（死亡/存命 + 利害関係開示）が最上部

    ### 5. 死亡/存命 hard rule

    `perspective-quickref` v2.1 は強制：
    - **故人**（10 名：munger / feynman / drucker / schopenhauer / musashi / da_vinci / sun-tzu / nietzsche / jung / jobs / meadows）—— 毎回生没年を能動宣言、2026 市場を観測したふりをしない
    - **存命**（9 名：buffett / dawkins / musk / naval / paul-graham / karpathy / hassabis / taleb / thiel）—— 2025+ 具体的立場を予測しない；GOOGL/crypto/政治的利害があれば明示

    ## できないこと（正直な境界）

    **重要** —— 「できること」より大切な部分：

    ### 1. LLM の幻覚は消せない

    Perspective skill は**思考スタイル模倣**であり、**事実データベース**ではない。バフェット perspective が出力する「intrinsic value $X」は** LLM 訓練データに基づく推定値**であり、Berkshire の内部評価モデルではない。

    **SOP**：具体的数字 / イベント / 予測は必ず verify（SOUL.md "Data Hygiene" ルールに従う）。

    ### 2. 「人物の能力範囲外」の助言はできない

    - drucker は 2002 年までインターネットを見ていない → **2026 年のテック起業アドバイスには不適**
    - musashi は 1645 年没 → **現代核兵器倫理の議論に使うな**
    - sun-tzu の生没年は不詳 → **「もし sun-tzu が 2026 露宇戦争を見たら」は予測するな**

    **SOP**：次元 4 の「能力範囲」は hard rule。違反 = 信頼崩壊。

    ### 3. 「利害関係バイアス」は解決できない

    - **hassabis の AGI タイムライン談** → GOOGL オプション利害を必ず宣言
    - **thiel の政治談** → MAGA 政治立場を必ず宣言
    - **musk の X プラットフォーム政策談** → X 利害を必ず宣言
    - **taleb の crypto 談** → 反体制立場を必ず宣言
    - **jobs の Apple/Disney 談** → 株主利害を必ず宣言

    **SOP**：v2.0 ルール —— いかなる商業 / 政治 / 投資利害 = honesty boundary に必ず含める。

    ### 4. 「未来を独立予測」はできない

    9 名の存命人物 —— 2025+ 具体的立場の予測はしない。例：
    - ❌ 「バフェットは 2027 年 AI をどう見るか」
    - ❌ 「musk の 2030 年火星計画進捗」
    - ❌ 「karpathy の GPT-6 に対する見解」

    **SOP**：人物の公的発言時間窓を超える予測 = 回答拒否。

    ### 5. 「self-catch」はできない（使用者 / 外部機構に依存）

    6/10 F-Kael 8 時間ループの教訓：**协作の堀 = 双方とも self-catch 能力を持つこと、相手側に catch を外注しないこと**。

    Perspective skill はツール —— 「今間違って使った」と能動的には言わない。self-catch は使用者（人間 / AI）が実行：
    - 時系列方向 self-check（「T-Xmin まもなく触发」前に必ず現在時刻を verify）
    - 数字 verify（具体数字 = 公式ソースを引く）
    - 利害関係開示（X perspective 使用前に X の利害バイアスを確認）
