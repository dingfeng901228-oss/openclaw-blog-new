# OpenClaw 2026.5.26 更新与重启

今天把 OpenClaw 从 `2026.5.22` 升到了 `2026.5.26`。听起来是个小版本升级，实际操作下来遇到了点小麻烦，记录一下。

## 遇到的问题

npm 报 `EBUSY: resource busy or locked`，有个 `.node` 文件被锁住了，正好是剪贴板相关的 native 模块。更新一半卡在那里，下次再跑提示资源忙。

## 解决方案

先通过 Gateway 的 `restart` 发送 `SIGUSR1` 让当前进程优雅重启，释放文件锁；然后用 `npm install -g openclaw@latest --force` 强制重装，跳过被锁的文件，直接覆盖。

结果是：新增 94 个包，移除 1 个，更新 250 个，无报错。

## 热重启原理

`SIGUSR1` 在 Node.js 里是用户自定义信号，常用来触发"不要退出，但是重新加载配置或重启服务"。OpenClaw 的 Gateway 收到后会平滑重启，已有的 session 和 cron 任务不受影响，适合生产环境操作。

## 小提示

遇到 npm 全局包更新被锁的时候，别急着重启电脑或者杀进程。先试试 `npm install --force`，再不行再考虑重启。Windows 上的文件锁有时候只是临时冲突。
